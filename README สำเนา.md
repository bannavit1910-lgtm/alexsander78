# Alexsander Store

เว็บขายไอดีเกม/สกิน พร้อมระบบสมัครสมาชิก, แดชบอร์ดผู้ใช้, ระบบเติมเงิน, และแอดมินพาแนลจัดการสินค้า/สมาชิก/คำสั่งซื้อ

## วิธีติดตั้งและรันบนเครื่องคุณ (VS Code)

1. เปิดโฟลเดอร์นี้ใน VS Code
2. เปิด Terminal แล้วรัน:
   ```
   npm install
   ```
3. คัดลอกไฟล์ `.env.example` เป็น `.env` แล้วแก้ค่าต่างๆ (อย่างน้อยต้องเปลี่ยน `SESSION_SECRET`)
   ```
   cp .env.example .env
   ```
4. รันเซิร์ฟเวอร์:
   ```
   npm start
   ```
   หรือระหว่างพัฒนาใช้ (auto-reload เมื่อแก้โค้ด):
   ```
   npm run dev
   ```
5. เปิดเบราว์เซอร์ไปที่ `http://localhost:3000`

## โครงสร้างโปรเจกต์

```
app.js                 จุดเริ่มต้นของเซิร์ฟเวอร์ Express
db.js                  ตั้งค่าฐานข้อมูล SQLite และสร้างตารางอัตโนมัติ
middleware/
  auth.js              ตรวจสอบการล็อกอิน / สิทธิ์แอดมิน
  upload.js            ตั้งค่าอัปโหลดรูปสินค้า (multer)
  rateLimit.js         จำกัดจำนวน request ต่อ IP แบบง่าย
routes/
  auth.js              สมัครสมาชิก / เข้าสู่ระบบ / ออกจากระบบ
  products.js          หน้าร้าน, ซื้อสินค้า
  orders.js            แดชบอร์ดผู้ใช้ (ดูประวัติคำสั่งซื้อ/เติมเงิน)
  topup.js             ส่งคำขอเติมเงิน
  admin.js             แอดมินพาแนลทั้งหมด (สินค้า/สมาชิก/คำสั่งซื้อ/เติมเงิน/log)
utils/discord.js        ส่งแจ้งเตือนไปยัง Discord ผ่าน Webhook
views/                  หน้าเว็บ (EJS templates)
public/                 CSS และรูปสินค้าที่อัปโหลด
data/store.db           ไฟล์ฐานข้อมูล SQLite (จะถูกสร้างอัตโนมัติ)
```

## ฟีเจอร์ที่มีอยู่แล้ว

- สมัครสมาชิก / เข้าสู่ระบบ (เข้ารหัสด้วย bcrypt, session-based auth)
- ระบบยศสมาชิก Member / VIP / MVP (แอดมินปรับได้จากหน้า "สมาชิก")
- แดชบอร์ดผู้ใช้: ยอดเงินคงเหลือ, ประวัติคำสั่งซื้อ, ประวัติการเติมเงิน
- หน้าร้าน: ค้นหา, กรองตามหมวดหมู่, แท็ก "ขายดี" และป้ายส่วนลด, สีตามระดับความหายาก (rare/epic/legend)
- ระบบเติมเงิน: ผู้ใช้ส่งคำขอ → แอดมินตรวจสอบและอนุมัติในหน้า "คำขอเติมเงิน" → ยอดเข้าบัญชีอัตโนมัติ
- แอดมินพาแนล: เพิ่ม/แก้ไข/ลบสินค้า พร้อมอัปโหลดรูป, ปรับราคาและส่วนลด, ดูคำสั่งซื้อทั้งหมด, กราฟยอดขายรายวัน 14 วันล่าสุด, Activity Log ของแอดมิน
- Rate limiting เบื้องต้นเพื่อลดความเสี่ยงจากการยิง request ถล่ม

## สิ่งที่ควรทำเพิ่มก่อนใช้งานจริงกับลูกค้าจริง

1. **เชื่อมต่อ TrueMoney Wallet (หรือผู้ให้บริการชำระเงินที่คุณสมัครใช้งานจริง) แบบอัตโนมัติ** — ตอนนี้ `routes/topup.js` มีฟังก์ชัน `verifyTruemoneyVoucher()` เป็นเพียง stub ที่ยังไม่เชื่อมต่อ API จริง ต้องนำ SDK/เอกสารทางการของผู้ให้บริการที่คุณสมัครมาใส่แทน พร้อมตรวจสอบลายเซ็น Webhook ตามเอกสารนั้น
2. **หน้าเปลี่ยนรหัสผ่าน / ลืมรหัสผ่าน** ยังไม่มีในเวอร์ชันนี้
3. **HTTPS และตั้งค่า cookie ให้ `secure: true`** เมื่อ deploy ขึ้น production จริง
4. **สำรองฐานข้อมูล (`data/store.db`) เป็นประจำ**
5. พิจารณาเปลี่ยน rate limiter จาก in-memory เป็น Redis หากมีมากกว่า 1 instance
